# 设计与决策（workflow design）

## 背景与目标

上一节信息量大的课（1~2 小时录音/妙记 + 课件 + 群消息），想用 AI 筛选、归纳成一份**可勾选的 todo list + 知识库**，做到"已掌握的勾掉，剩下的就是该专注学的"，并把作业/练习沉淀下来。

原始需求拆为三个维度：① 这节课讲了哪些知识点？② 有哪些课件需要看？③ 群里大家踩了哪些坑？

本项目覆盖 **第一部分（课堂）**：课件 + 课堂记录。第二部分（群聊分析 + 定时通知）为后续增强，不在本仓库范围内。

## 架构与数据流

```
[输入]
  课堂记录文档(妙记/转写导出) + 课件(HTML幻灯片 + DOC正文)
     │  docs +search 定位 → docs +fetch 读 markdown 正文
     │  HTML课件: drive +download → Chrome无头逐页截图 → 幻灯片PNG
     ▼
[AI 处理]  按知识点归纳，每条知识点关联:
     │     ① 整段课堂原文(标时间区间) ② 对应幻灯片页(若有)
     ▼
[输出]
  ① 单课文档(每条知识点 + 原文 + 幻灯片图)
  ② 知识库主文档(跨课汇总，管勾选状态)
     │  docs +create 建文本框架 → media-insert 上传图 → block_move_after 把图插到对应知识点下
     ▼
[用户]  在主文档勾掉已掌握的 → 剩下即为该专注项
```

## 产出物结构

### ① 单课文档（每节课一篇，例：「Day3 学习卡」）
每条知识点自带"原文+配图"，不单列课件索引：
1. **知识点复选框**：飞书原生 todo，默认未勾选，一句话概括
2. **整段课堂原文**：`<callout>` 灰色框，标时间区间(如 `01:12~03:34`)，**写整段而非一句话**
3. **课件幻灯片图**(若有对应页)：HTML课件截图插入，标注页码
   - 无对应 PPT 页(纯口述)：只放原文，注明"课件无对应页"
   - HTML缺图、DOC有补充：从 DOC 课件补原文，标注章节而非页码

顶部 callout 附课堂记录链接 + 该 Day 的**全部**课件超链接。

### ② 知识库主文档（全程一篇，累积）
- 按 Day 分章节，只放跨课的待学知识点 todo（精简版，不含原文/图）
- 勾选状态在此统一管理（一屏看全"还有哪些没学"）
- 每条 todo 链接回对应单课文档

## 关键决策

- **文档组织**：单课文档（详细+配图）+ 知识库主文档（跨课汇总+管勾选）。兼顾"单课详情"与"知识库统览"。
- **初始勾选**：所有知识点默认未勾选，由用户自己 check。"哪些已会"只能用户自己判断，第一版最简单、不会误判。
- **勾选落地**：用飞书原生 todo block，状态存飞书侧；workflow 重跑时先读主文档已勾选项、保留不覆盖，只追加新内容（幂等更新）。

## 非目标（YAGNI）

- 不做群聊分析、通知推送（后续增强）
- 不做 AI 预判已掌握/未掌握（用户明确要自己判断）
- 不做音视频转写（用导出的文字记录替代）
- 第一版不做定时自动推送
